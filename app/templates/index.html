How DBSC Works

  1. Login sets a short-lived cookie (600s) and sends Secure-Session-Registration header
  2. Chrome generates a TPM-bound key pair and registers it via POST /dbsc/start
  3. When the cookie expires, Chrome automatically refreshes it via POST /dbsc/refresh
  4. Refresh proves device possession via signed JWT — stolen cookies cannot be refreshed from another device
